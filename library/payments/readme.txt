  Каждой платёжной системе соответствует своя папка. В папке должно быть 3 обязательные файла: 1)pay_name.cfg, 2)pay_name.class.php, 3)icon.gif, где pay_name - название платёжной системы.

  Файл pay_name.cfg представляет собой ini-файл, в котором перечисляются значения, необходимые для подключения системы. Формат файла такой:
  param_name1 = description1  ; comment
  param_name2 = description2  ; comment
  ...
Эти значения будут добавлены в форму настройки платёжной системы в админке. У каждой платёжной системы могут быть свои, соответствующие её настройки.

  Файл pay_name.class.php - основной класс.

  Файл icon.gif - иконка платёжной системы, которая выводится на странице оплаты.

  Форма настройки платёжных систем в админке создаётся динамически. В настройках выводятся только те системы, папки которых есть в payments. Это значит, если клиенту нужно поставить возможность подключения только к robokassa и interkassa, оставляются только папки соответствующие этим платёжным системам. Настройки других платёжных систем пользователь в админке не увидит.
